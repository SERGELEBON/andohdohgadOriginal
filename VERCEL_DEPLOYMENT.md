## Troubleshooting

### Si le build échoue sur Vercel:

1. **Clear Build Cache**
   - Settings → General → Clear Build Cache

2. **Vérifier Node Version**
   - `.nvmrc` doit contenir: `20`

3. **Vérifier les variables d'environnement**
   - Toutes les variables `VITE_*` doivent être définies

4. **Redéployer**
   - Deployments → Latest → ... → Redeploy

### Erreurs TypeScript

Si erreur `Cannot find module '@/lib/supabase/client'`:
- Le fichier existe à: `src/lib/supabase/client.ts`
- L'alias `@` est configuré dans `vite.config.ts` et `tsconfig.json`
- Build locale fonctionne → problème de cache Vercel

**Solution**: Clear cache + Redeploy

## Post-Deployment

Après le premier déploiement réussi:

1. **Mettre à jour VITE_APP_URL**
   - Remplacer par votre vraie URL Vercel
   - Redéployer

2. **Configurer Supabase**
   - Authentication → URL Configuration
   - Site URL: `https://your-domain.vercel.app`
   - Redirect URLs: `https://your-domain.vercel.app/**`
